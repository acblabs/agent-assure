# agent-assure v0.4.4

v0.4.4 is a package release for the completed Sprint 12 process-assurance
positioning work. It keeps the persisted artifact schema at `0.4.3` and uses
the committed `schemas/v0.4.3` snapshot.

## Added

- Added `docs/measurement/process_assurance_vs_answer_quality_eval.md` to
  explain how answer-quality evaluation and observable process-preservation
  checks complement each other.
- Added the bundled `process_measurement_cases` fixture suite and
  `agent-assure demo measurement-cases` for offline review of same-answer
  process changes.
- Extended evidence-diff HTML process tables with operational counters,
  measured usage summaries, and usage deltas.

## Changed

- Tightened control coverage mapping semantics and documentation for
  packet-resident control-specific evidence, scoped mapping conditions, mapping
  resource loading, and review-side failure states.
- Hardened claim-boundary checks for ATLAS/MITRE wording while retaining
  approved limitation sentences.
- Kept package version `0.4.4` mapped to schema version `0.4.3` in the release
  tag validator because this release does not introduce a new persisted JSON
  artifact shape.

## Boundary

The process-measurement cases are deterministic fixture examples, not
comparative product measurements. Evidence-diff usage and process signals are
review evidence; they do not rank answer quality, framework conformance,
production safety, or business outcomes.
