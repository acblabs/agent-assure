# Agentic Measurement Use Case

This measurement use case describes a narrow problem: deterministic
governance-pipeline changes can alter evidence linking and control behavior even
when fixed model and tool fixtures keep the visible task inputs stable.

The repository evaluates compiled expectations, built-in deterministic controls,
fixture equivalence, and provenance changes for local fixture suites. These
results are not live model-quality, stochastic reliability, safety,
certification, compliance, or endorsement evidence.

## Reproducible Result Tables

The following tables are derived from local fixture artifacts generated by the
suite compile, suite run, evaluate, and compare commands in this repository.
The prior-authorization suite contains ten fixed cases; the expense-approval
suite contains three fixed cases.

## Flagship Showcase Draft

The linkable showcase is the prior-authorization evidence-normalization
candidate. The baseline and candidate use the same compiled suite and fixture
manifest. The baseline passes all ten fixed cases. The candidate keeps the
visible answer stable for the shared-source multi-claim case
(`recommendation=approve; outcome=approve`) but loses the structured evidence
link for the fixture-declared material claim `claim-duration`.

The expected candidate finding is:

```text
case_id: shared-source-multi-claim
control_id: material_claims_have_evidence
target: claim:claim-duration
reason_code: MATERIAL_CLAIM_MISSING_EVIDENCE
```

The baseline-to-candidate comparison report classifies the result as
`new_failure` with fixture-equivalence state `pass`. The generated report is
human-reviewable because it shows the failing case, control, target, reason
code, missing evidence link, fixture-equivalence result, behavioral evidence
reference change, and separate provenance changes. Reproduction commands,
expected output fields, workflow snippet, and artifact digests are maintained in
`docs/showcase.md`.

## Failure Analysis Examples

The evidence-normalization candidate introduces one deterministic failure: a
fixture-declared material claim loses its structured evidence link in the
shared-source multi-claim case. The ordinary one-source cases remain passing,
which keeps the failure localized to the intended evidence-linking invariant.

The provider-policy candidates demonstrate governance-control failures rather
than model-quality findings. Their reports identify outcome, human-review, and
review-boundary controls that fail under fixed fixtures.

Provenance changes are reported separately from verdicts. A configuration digest
change across a candidate variant helps reviewers understand what changed, but
it is not counted as a regression unless an expectation, invariant, policy, or
configured gate fails.
